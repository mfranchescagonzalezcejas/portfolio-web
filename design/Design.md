# DevDigi Portfolio Redesign

This document defines the redesign direction for the current **DevDigi** portfolio website.

DevDigi is the personal developer brand of **Mercedes Franchesca Gonzalez Cejas**, a **Mobile Developer / Software Engineer** focused on Flutter, Android/Kotlin, iOS/Swift, clean architecture, REST APIs, CI/CD, QA, and production mobile apps.

The goal is to make the portfolio feel more premium, polished, recruiter-friendly, and visually impressive while keeping every claim realistic, truthful, and professional.

## Positioning

| Topic                   | Direction                                                                               |
| ----------------------- | --------------------------------------------------------------------------------------- |
| Brand                   | DevDigi                                                                                 |
| Person behind the brand | Mercedes Franchesca Gonzalez Cejas                                                      |
| Role                    | Mobile Developer / Software Engineer                                                    |
| Focus                   | Flutter, Android, iOS, clean architecture, REST APIs, CI/CD, QA, production mobile apps |
| Brand type              | Personal developer brand, not an agency                                                 |

Required brand line:

> DevDigi is my personal developer brand — a space where I showcase my mobile work, projects and technical growth.

## Visual Direction

- Premium dark mobile-engineering portfolio.
- Deep navy / near-black background.
- Teal, cyan, and soft blue accents.
- Elegant gradients.
- Soft borders and subtle glow effects.
- Glassmorphism only in selected cards.
- Mobile app mockups.
- Code snippet cards.
- Strong visual hierarchy.
- Recruiter-friendly and professional tone.
- Not a fake agency website.
- Not a generic SaaS landing page.
- Not too neon or cyberpunk.

## Global Requirements

- Replace all “Hire me” buttons with “Contact me”.
- Make DevDigi clearly feel like a personal developer brand.
- Improve spacing so the site feels premium but slightly more compact.
- Make the mobile responsive version feel polished, like a high-quality mobile product landing page.
- Keep the website fast and lightweight.
- Make it fully responsive.
- Improve accessibility.
- Use strong visual hierarchy.
- Use subtle animations, not excessive motion.
- Keep the dark teal/blue DevDigi identity.
- Do not invent metrics.
- Do not invent companies.
- Do not claim ownership of professional apps.
- Do not expose confidential implementation details.
- Keep everything realistic, professional, and recruiter-friendly.
